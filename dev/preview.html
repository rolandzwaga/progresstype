ProgressType

An OpenType variable font that renders {h:25,10,15} as a stacked horizontal progress bar — and {v:75} as a vertical one.

Single-segment (fixed-width)

Bar is a fixed-size container; fill grows inside it. The empty space on the right is the unfilled portion of the track.

CPU{h:23}23%
Memory{h:67}67%
Disk{h:91}91%
Network{h:8}8%

Stacked multi-segment (fixed-width via padding)

Multi-segment bars are variable-width by default — they grow with the sum of values. To make them appear fixed-width, pad the values to total 100 and use a CSS @font-palette-values override so the trailing position uses the track color. The empty-looking tail is just a normal segment painted in the track color.

CPU+IO 70% {h:30,40,30} 30 + 40 + (30 pad)
3 phases 50% {h:25,10,15,50} 25 + 10 + 15 + (50 pad)
4 phases 70% {h:20,15,10,25,30} 20 + 15 + 10 + 25 + (30 pad)
7 even, 30% gap {h:10,10,10,10,10,10,10,30} 7×10 + (30 pad)

Naturally totals 100% — no padding needed

When your data already sums to 100, every segment is colored and the bar fills the full width. RAG status is the canonical example.

Status{h:45,30,25}on track / at risk / blocked
/* Trailing-track palette: paint the Nth position transparent so only the
   strip + border outline shows through — matches how the empty portion of
   single-segment {h:NN} looks. */
@font-palette-values --pad4 {
  font-family: 'ProgressType';
  base-palette: 2;                /* categorical */
  override-colors: 4 rgba(0,0,0,0);
}

.bar { font-family: 'ProgressType', monospace; font-feature-settings: 'liga' 1, 'calt' 1; }
.bar.padded { font-palette: --pad4; }

<p class="bar padded">{h:25,10,15,50}</p>

Custom palettes via CSS

Define your own palettes with @font-palette-values and override-colors — no font rebuild needed.

brand {h:25,15,20,10,15}
earth {h:12,12,12,12,12,12,12,12}
design tokens {h:30,25,20} 
/* Override individual palette entries — start from a shipped base palette */
@font-palette-values --brand {
  font-family: 'ProgressType';
  base-palette: 2;
  override-colors:
    0 #1f2937,        /* track */
    1 #6366f1,        /* indigo  (1st segment)  */
    2 #ec4899,        /* pink    (2nd segment)  */
    3 #f59e0b;        /* amber   (3rd segment)  */
}

/* You can also pull from CSS custom properties — connects to design tokens */
@font-palette-values --tokens {
  font-family: 'ProgressType';
  base-palette: 0;
  override-colors:
    1 var(--brand-primary),
    2 var(--brand-secondary),
    3 var(--brand-accent);
}

.bar { font-palette: --brand; }
<p class="bar">{h:25,15,20}</p>

Heads-up: the standard CSS color property does not recolor CPAL fonts — only font-palette does. Browser support: Chrome 101+, Safari 15.4+, Firefox 107+ (~94% globally). Older renderers fall back to the base-palette colors instead of honoring overrides.

Usage

@font-face {
  font-family: 'ProgressType';
  src: url('ProgressType[wdth,wght].woff2') format('woff2');
  font-weight: 100 900;
  font-stretch: 50% 150%;
}

@font-palette-values --rag {
  font-family: 'ProgressType';
  base-palette: 1;       /* 0=sequential, 1=rag, 2=categorical, 3=ocean, 4=mono */
}

.bar {
  font-family: 'ProgressType', monospace;
  font-feature-settings: 'liga' 1, 'calt' 1;
  font-palette: --rag;
}

<p class="bar">Status: {h:45,30,25}</p>
